# Флешкарточки 🧠

PWA-приложение с карточками и **интервальными повторениями** (алгоритм SM-2, как в Anki).
Один код работает в браузере на Linux и на iPhone (можно установить на экран «Домой»).
Колоды хранятся в этом репозитории и **обновляются из GitHub** — поправил файл, запушил, приложение подхватило.

## Возможности

- Вопрос на одной стороне, ответ — на обороте (переворот по клику/пробелу).
- Оценки **Не помню / Трудно / Хорошо / Легко** планируют следующий показ карточки.
- **Несколько колод** — выбор колоды в шапке; у каждой свой прогресс и дневной лимит.
- Режим **«Пройти все заново»** и список **«Стоит повторить»** (что отметил «не помню»/«трудно»).
- Ответ-список (`1. … 2. …`) выводится нумерованным списком.
- Работает офлайн (service worker кэширует приложение и колоды).
- Прогресс изучения хранится локально на устройстве (localStorage).
- Горячие клавиши: `Пробел` — перевернуть, `1`–`4` — оценки.

## Структура колод

Список колод описан в манифесте `decks.json`, а карточки лежат в папке `decks/`:

```json
// decks.json
{
  "decks": [
    { "id": "network", "name": "Сеть и HTTP", "file": "decks/network.json" }
  ]
}
```

```json
// decks/network.json — карточки колоды
[
  { "q": "Вопрос 1", "a": "Ответ 1" },
  { "q": "Вопрос 2", "a": "Ответ 2" }
]
```

В карточках также понимаются ключи `question`/`answer`, `front`/`back` и объект вида `{ "Вопрос": "Ответ" }`.
Если ответ оформить как `1. … 2. … 3. …`, он покажется нумерованным списком.

## Локальный запуск (Linux)

PWA с модулями нельзя открывать через `file://` — нужен локальный сервер:

```bash
cd flashcards
python3 -m http.server 8000
# открой http://localhost:8000
```

## Публикация на GitHub Pages (доступ с iPhone)

1. Создай репозиторий на GitHub и запушь туда этот проект:
   ```bash
   git remote add origin https://github.com/<юзер>/<репо>.git
   git push -u origin main
   ```
2. В репозитории: **Settings → Pages → Build and deployment → Source: Deploy from a branch**,
   ветка `main`, папка `/ (root)`. Сохрани.
3. Через минуту приложение будет доступно по адресу
   `https://<юзер>.github.io/<репо>/`.

### Установка на iPhone

Открой адрес в **Safari** → кнопка «Поделиться» → **«На экран Домой»**.
Появится иконка-приложение, работающее в полный экран и офлайн.

## Как добавлять карточки

Редактируй нужный файл колоды в `decks/` (на компьютере или прямо на github.com), коммить и пушь.
В приложении нажми ⟳ — подтянется свежая колода. Прогресс по старым карточкам сохранится
(карточка опознаётся по тексту вопроса в пределах своей колоды).

## Как добавить новую колоду

1. Создай файл, например `decks/javascript.json`, с карточками.
2. Добавь запись в `decks.json`:
   ```json
   { "id": "javascript", "name": "JavaScript", "file": "decks/javascript.json" }
   ```
3. Закоммить и запушь. В приложении нажми ⟳ — новая колода появится в списке выбора.

`id` должен быть уникальным (по нему хранится прогресс колоды).

## Настройки (⚙ в приложении)

- **Ссылка на манифест колод** — по умолчанию `./decks.json`. Можно указать чужую raw-ссылку:
  `https://raw.githubusercontent.com/<юзер>/<репо>/main/decks.json`.
- **Новых карточек в день** — лимит свежих карточек за сутки на каждую колоду (как в Anki).
- **Сбросить прогресс** — обнуляет расписание текущей колоды (карточки остаются).

## Иконки

Сгенерированы скриптом без зависимостей. Пересоздать:

```bash
python3 tools/gen_icons.py
```

## Обновление приложения

Если меняешь HTML/CSS/JS — подними `VERSION` в `sw.js`, чтобы у пользователей обновился кэш.

## Структура

```
index.html              оболочка приложения
css/styles.css          стили
js/srs.js               алгоритм интервальных повторений (SM-2)
js/app.js               логика приложения и UI
sw.js                   service worker (офлайн + обновление колод)
manifest.webmanifest    манифест PWA
decks.json              список колод (манифест)
decks/                  файлы колод с карточками
icons/                  иконки приложения
tools/gen_icons.py      генератор иконок
```
