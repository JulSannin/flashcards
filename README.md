# Флешкарточки 🧠

PWA-приложение с карточками и **интервальными повторениями** (алгоритм SM-2, как в Anki).
Один код работает в браузере на Linux и на iPhone (можно установить на экран «Домой»).
Колода карточек хранится в этом репозитории и **обновляется из GitHub** — поправил файл, запушил, приложение подхватило.

## Возможности

- Вопрос на одной стороне, ответ — на обороте (переворот по клику/пробелу).
- Оценки **Не помню / Трудно / Хорошо / Легко** планируют следующий показ карточки.
- Колода читается из `cards.json` (можно указать любую raw-ссылку GitHub в настройках).
- Работает офлайн (service worker кэширует приложение и последнюю колоду).
- Прогресс изучения хранится локально на устройстве (localStorage).
- Горячие клавиши: `Пробел` — перевернуть, `1`–`4` — оценки.

## Формат файла с карточками — `cards.json`

```json
[
  { "q": "Вопрос 1", "a": "Ответ 1" },
  { "q": "Вопрос 2", "a": "Ответ 2" }
]
```

Также понимаются ключи `question`/`answer`, `front`/`back` и объект вида `{ "Вопрос": "Ответ" }`.

## Локальный запуск (Linux)

PWA с модулями нельзя открывать через `file://` — нужен локальный сервер:

```bash
cd flashcards
python3 -m http.server 8000
# открой http://localhost:8000
```

## Публикация на GitHub Pages (доступ с iPhone)

1. Создай репозиторий на GitHub и запушь туда этот проект:
   ```bash
   git remote add origin https://github.com/<юзер>/<репо>.git
   git push -u origin main
   ```
2. В репозитории: **Settings → Pages → Build and deployment → Source: Deploy from a branch**,
   ветка `main`, папка `/ (root)`. Сохрани.
3. Через минуту приложение будет доступно по адресу
   `https://<юзер>.github.io/<репо>/`.

### Установка на iPhone

Открой адрес в **Safari** → кнопка «Поделиться» → **«На экран Домой»**.
Появится иконка-приложение, работающее в полный экран и офлайн.

## Как добавлять карточки

Просто редактируй `cards.json` (на компьютере или прямо на github.com), коммить и пушь.
В приложении нажми ⟳ (или потяни заново) — подтянется свежая колода.
Новые карточки появятся, прогресс по старым сохранится (карточка опознаётся по тексту вопроса).

## Настройки (⚙ в приложении)

- **Ссылка на файл с карточками** — по умолчанию `./cards.json`. Можно указать чужую raw-ссылку:
  `https://raw.githubusercontent.com/<юзер>/<репо>/main/cards.json`.
- **Новых карточек в день** — лимит свежих карточек за сутки (как в Anki).
- **Сбросить прогресс** — обнуляет расписание (карточки остаются).

## Иконки

Сгенерированы скриптом без зависимостей. Пересоздать:

```bash
python3 tools/gen_icons.py
```

## Обновление приложения

Если меняешь HTML/CSS/JS — подними `VERSION` в `sw.js`, чтобы у пользователей обновился кэш.

## Структура

```
index.html              оболочка приложения
css/styles.css          стили
js/srs.js               алгоритм интервальных повторений (SM-2)
js/app.js               логика приложения и UI
sw.js                   service worker (офлайн + обновление колоды)
manifest.webmanifest    манифест PWA
cards.json              колода карточек
icons/                  иконки приложения
tools/gen_icons.py      генератор иконок
```
